Settings

Name Default Args Description
element string The component element.
settings string The component settings.
settings.id `contextual-action-modal-cnt` string The id to use for the CAP, or defaults to generated.
settings.buttons null array A list of buttons that will sit in the toolbar's Buttonset area.
settings.title 'Contextual Action Panel' string String that sits in the toolbar's title field.
settings.content null content Pass content through to CAP.
settings.initializeContent true boolean Initialize content before opening with defaults.
settings.trigger 'click' string Can be 'click' or 'immediate'.

Code Example

This example shows how to invoke a Contextual Action Panel (CAP) and pass in the content for the CAP body. The buttons option lets you customize the CAPâ€™s toolbar and functions.


$('body').contextualactionpanel({
  id: 'contextual-action-modal-id',
  title: 'Expenses: $50,000.00',
  content: '<markup>',
  trigger: 'immediate',
  buttons: [
    {
      type: 'input',
      text: 'Keyword',
      id: 'filter',
      name: 'filter',
      cssClass: 'searchfield'
    }, {
      text: 'Close',
      cssClass: 'btn',
      icon: '#icon-close'
    }
  ]
});

Keyboard Shortcuts

The Contextual Action Panel inherits the following keystrokes from other controls:

  • Escape closes the Contextual Action Panel the same way it would close an Input Dialog.
  • When focused on elements inside the Toolbar at the top of the Contextual Action Panel, all Toolbar keystrokes take effect.
  • Tab - should never tab off the Dialog

States and Variations

  • Hidden
  • Visible

Responsive Guidelines

The Contextual Action Panel responds similarly to an Input Dialog, in that it will resize and center itself horizontally and vertically in the browser window, never fitting beyond 80% width and height of the window.

Upgrading from 3.X

  • Did not exist in 3.X, but any complex Modals should be changed to use this.
