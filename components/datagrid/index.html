API Details

Settings

  • actionableMode Boolean  -  If actionableMode is "true”, tab and shift tab behave like left and right arrow key, if the cell is editable it goes in and out of edit mode. F2 - toggles actionableMode "true" and "false"
  • cellNavigation Boolean  -  If cellNavigation is "false”, will show border around whole row on focus
  • rowNavigation Boolean  -  If rowNavigation is "false”, will NOT show border around the row
  • alternateRowShading Boolean  -  Sets shading for readonly grids
  • columns Array  -  an array of columns (see column options)
  • dataset Array  -  an array of data objects
  • columnReorder Boolean  -  Allow Column reorder
  • saveColumns Boolean  -  Save Column Reorder and resize
  • saveUserSettings Object  -  Save one or all of the following to local storage : columns: true, rowHeight: true, sortOrder: true, pagesize: true, activePage: true, filter: true
  • editable Boolean  -  Enable editing in the grid, requires column editors.
  • isList Boolean  -  Makes the grid have readonly "list" styling
  • menuId String  -  Id to link a right click context menu element
  • menuSelected String  -  Callback for the grid level context menu
  • uniqueId String  -  Unique ID to use as local storage reference and internal variable names
  • rowHeight String  -  Controls the height of the rows / number visible rows. May be (short, medium or normal)
  • selectable String  -  Controls the selection Mode this may be: false, 'single' or 'multiple' or 'mixed'
  • groupable Object  -  Controls fields to use for data grouping Use Data grouping fx. {fields: ['incidentId'], supressRow: true, aggregator: 'list', aggregatorOptions: ['unitName1']}
  • clickToSelect Boolean  -  Controls if using a selection mode if you can click the rows to select
  • toolbar Object  -  Toggles and appends toolbar features fx.. {title: 'Data Grid Header Title', results: true, keywordFilter: true, filter: true, rowHeight: true, views: true}
  • initializeToolbar Boolean  -  Set to false if you will initialize the toolbar yourself
  • paging Boolean  -  Enable paging mode
  • pagesize Number  -  Number of rows per page
  • pagesizes Array  -  Array of page sizes to show in the page size dropdown.
  • indeterminate Boolean  -  Disable the ability to go to a specific page when paging.
  • source Function  -  Callback function for paging
  • hidePagerOnOnePage Boolean  -  If true, hides the pager if there's only one page worth of results.
  • filterable Boolean  -  Enable Column Filtering, This will require column filterTypes as well.
  • disableClientFilter Boolean  -  Disable Filter Logic client side and let your server do it
  • disableClientSort Boolean  -  Disable Sort Logic client side and let your server do it
  • resultsText String  -  Can provide a custom function to adjust results text on the toolbar
  • rowReorder Boolean  -  If set you can reorder rows. Requires rowReorder formatter
  • showDirty Boolean  -  If true the dirty indicator will be shown on the rows
  • allowOneExpandedRow Boolean  -  Controls if you cna expand more than one expandable row.
  • enableTooltips Boolean  -  Process tooltip logic at a cost of performance
  • disableRowDeactivation Boolean  -  if a row is activated the user should not be able to deactivate it by clicking on the activated row
  • sizeColumnsEqually Boolean  -  If true make all the columns equal width
  • expandableRow Boolean  -  If true we append an expandable row area without the rowTemplate feature being needed.
  • redrawOnResize Boolean  -  If set to false we skip redraw logic on the resize of the page.

Functions

addRow

Add a row of data to the grid and dataset.

Parameters

  • data Object  -  An data row object
  • location String  -  Where to add the row. This can be 'top' or leave off for 'bottom'

removeRow

Remove a row of data to the grid and dataset.

Parameters

  • row Number  -  The row index
  • nosync Boolean  -  Dont sync the selected rows.

removeSelected

Remove all selected rows from the grid and dataset.

clearFilter

Clear the Filter row Conditions and Reset the Data.

hideColumn

Hide a column.

Parameters

  • id String  -  The id of the column to hide.

Code Example

The Datagrid is created via JS or via direct html markup (css table styling). You should pass an array of objects in to the grid on the dataset object. Also pass the columns array for the column information. There are a number of events described in the events and api section, as well as the column settings.

A Read-Only Datagrid uses "Formatters" to render cell content. A number of these are listed in the api section, and it is possible to create your own.



<div class="row">
  <div class="twelve columns">
    <div id="datagrid">
    </div>
  </div>
</div>



  //Define Columns for the Grid.
  columns.push({ id: 'productId', name: 'Product Id', field: 'productId', width: '50%', formatter: Formatters.Readonly});
  columns.push({ id: 'productName', name: 'Product Name', sortable: false, field: 'productName', width: '50%', formatter: Formatters.Hyperlink});

  //Init and get the api for the grid
  $('#datagrid').datagrid({
    columns: columns,
    dataset: data //Json Array
  });

Keyboard Shortcuts

  • Tab  The initial tab enters the grid with focus on the first cell of the first row, often a header. A second tab moves out of the grid to the next tab stop on the page. Once focus is established in the grid, a TAB into or a Shift Tab into the grid will return to the cell which last had focus.
  • Left and Right Arrow Move focus to the adjacent column's cell. There is no wrap at the end or beginning of columns.
  • Up and Down Arrow Move focus to the adjacent row's cell. There is no wrap at the first or last row.
  • Home moves focus to the first cell of the current row
  • End moves focus to the last cell of the current row
  • Page Up moves focus to the first cell in the current column
  • Page Down moves focus to the last cell in the current column
  • Enter toggles edit mode on the cell if it is editable. There is also an "auto edit detection". If the user starts typing then edit mode will happen automatically without enter.
  • F2 toggles actionable Mode. Pressing the Tab key while in actionable mode moves focus to the next actionable cell. There is no wrap at the end or beginning of columns. While in actionable mode you can do things like type + enter. This will move you down a row when you hit enter. If the cell has a control that uses down arrow (like the dropdowns or lookups that are editable). Then the user needs to hit enter to enable the edit mode on that cell.
