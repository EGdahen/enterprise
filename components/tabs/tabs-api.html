Settings

Name Type Description
addTabButton boolean If set to true, creates a button at the end of the tab list that can be used to add an empty tab and panel
addTabButtonCallback if defined as a function, will be used in-place of the default Tab Adding method
appMenuTrigger boolean If set to true, will force an App Menu trigger to be present on Non-Vertical Tabs implementatations.
ajaxOptions if defined, will be used by any internal Tabs AJAX calls as the desired request settings.
beforeActivate If defined as a function, fires this before a tab is activated to allow a possible "veto" of the tab swap (SOHO-5250).
containerElement Defines a separate element to be used for containing the tab panels. Defaults to a .tab-panel-container element that is created if it doesn't already exist.
changeTabOnHashChange boolean If true, will change the selected tab on invocation based on the URL that exists after the hash.
hashChangeCallback If defined as a function, provides an external method for adjusting the current page hash used by these tabs.
lazyLoad boolean if true, when using full URLs in tab HREFs, or when using Ajax calls, tabs will be loaded as needed instead of the markup all being established at once.
moduleTabsTooltips boolean if true, will display a tooltip on Module Tabs with cut-off text content.
source If defined, will serve as a way of pulling in external content to fill tabs.
sourceArguments object If a source method is defined, this flexible object can be passed into the source method, and augmented with parameters specific to the implementation.
tabCounts boolean If true, Displays a modifiable count above each tab.
verticalResponsive boolean If Vertical Tabs & true, will automatically switch to Horizontal Tabs on smaller breakpoints.
