Settings

Name Type Description
steps number The number of steps to show.
completed number The number of steps complete (linear).
inProgress number The number of the in progress step (linear).
iconType boolean The icon to display fx. 'icon-error', 'icon-confirm'
completedText string The completed text or uses a localized 'N of N Steps complete'. You can use {0} and {1} to replace n of n in the string.
extraText boolean The additional text to show on the right. Defaults to none. You can use {0} to replace with the steps remaining count and {1} to replace the number of steps.
completedColor string The color to show completed steps. Defaults to primary color.
allCompletedColor string The color to steps when all are completed. Defaults to primary color.
inProgressColor string The color to show in-progress steps. Defaults to ruby02.

Behavior Guidelines

  • When all steps are complete the chart shows in positive color (green) with an icon.
  • In progress steps shown in a ruby color

Code Example

You can initialize a step chart by putting a simple div in the page with data-options which contain any of the above options.


<div class="step-chart" data-options="{steps: 7, completed: 2, inProgress: 3, iconType: 'icon-error', extraText: '2 Days Overdue'}">
</div>

Or you can initialize the component using javascript, remove the data-options and initialize like a normal plugin.


$('#manual').stepchart({
  completed: 2,
  steps: 10,
  completedText: 'Amount Completed'
});

States and Variations

A Step Chart can take the following states:

  • Error
  • N Complete
  • All Complete

Responsive Guidelines

  • The chart adapts to the parent size and the steps will resize to equal portions.
