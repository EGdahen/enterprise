Settings

Name Default Args Description
element string The component element.
settings string The component settings.
settings.legend string Pass in custom markdown for the legend structure.
settings.legendKey string Key to use for the legend matching
settings.dataset [] string Hierarchical Data to display
settings.newData [] boolean New data to be appended into dataset
settings.templateId string Additional product name information to display
settings.mobileView false boolean If true will only show mobile view, default using device info.
settings.leafHeight null number Set the height of the leaf
settings.leafWidth null number Set the width of the leaf
settings.beforeExpand null string A callback that fires before node expansion of a node.
settings.paging false boolean If true show pagination.

Code Example

This example shows how to invoke the hierarchy component passing in data and legend info for matching.


    var legendData = [
      { 'value' : 'FT', 'label' : 'Full Time'     },
      { 'value' : 'PT', 'label' : 'Part Time'     },
      { 'value' : 'C',  'label' : 'Contractor'    },
      { 'value' : 'O',  'label' : 'Open Position' }
    ];

    $('#hierarchy').hierarchy({
      templateId: 'hierarchyChartTemplate',
      legendKey: 'EmploymentType',
      legend: legendData,
      dataset: data
    });
