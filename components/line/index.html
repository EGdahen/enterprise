API Details

Settings

Name Default Args Description
element string The plugin element for the constuctor
settings string The settings element.
settings.dataset [] array The data to use in the line/area/bubble.
settings.tooltip A custom tooltip or tooltip renderer function for the whole chart.
settings.isArea string Render as an area chart.
settings.isBubble false string Render as a bubble chart.
settings.showLegend true string If false the label will not be shown.
settings.xAxis object A series of options for the xAxis
settings.xAxis.rotate number Rotate the elements on the x axis. Recommend -65 deg but this can be tweaked depending on look.
settings.yAxis object A series of options for the yAxis
settings.xAxis.ticks object Data to control the number of ticks and y axis format. For example `{number: 5, format: ',.1s'}` would show only 5 yaxis points and format the data to show 1K, 1M, 1G ect.. This uses the d3 formatter.
settings.xAxis.formatText function A function that passes the text element and a counter. You can return a formatted svg markup element to replace the current element. For example you could use tspans to wrap the strings or color them.
settings.yAxis object A series of options for the yAxis
settings.yAxis.formatter function A d3 formatter for the yAxis points.
settings.hideDots false boolean If true no dots are shown
settings.axisLabels array Option to a label to one of the four sides. For Example `{left: 'Left axis label', top: 'Top axis label', right: 'Right axis label', bottom: 'Bottom axis label'}`
settings.animate true|false - will do or not do the animation. 'initial' will do only first time the animation.
settings.redrawOnResize true boolean If true, the component will not resize when resizing the page.
settings.dots object Option to customize the dot behavior. You can set the dot size (radius), the size on hover and stroke or even add a custom class. Example `dots: { radius: 3, radiusOnHover: 4, strokeWidth: 0, class: 'custom-dots'}`
settings.formatterString string Use d3 format some examples can be found on http://bit.ly/1IKVhHh
settings.emptyMessage object An empty message will be displayed when there is no chart data. This accepts an object of the form emptyMessage: `{title: 'No Data Available', info: 'Make a selection on the list above to see results', icon: 'icon-empty-no-data', button: {text: 'xxx', click: } }` Set this to null for no message or will default to 'No Data Found with an icon.'

Code Example

This example shows how to invoke the line chart option in the charts component. In this example we pass a dataset with x and y axis values for three lines.


var dataset = [{
  data: [{
      name: 'Jan',
      value: 12
  }, {
      name: 'Feb',
      value: 11
  }, {
      name: 'Mar',
      value: 14
  }, {
      name: 'Apr',
      value: 10
  }, {
      name: 'May',
      value: 14
  }, {
      name: 'Jun',
      value: 8
  }],
  name: 'Component A'
}, {
  data: [{
      name: 'Jan',
      value: 22
  }, {
      name: 'Feb',
      value: 21
  }, {
      name: 'Mar',
      value: 24
  }, {
      name: 'Apr',
      value: 20
  }, {
      name: 'May',
      value: 24
  }, {
      name: 'Jun',
      value: 28
  }],
  name: 'Component B'
}, {
  data: [{
      name: 'Jan',
      value: 32
  }, {
      name: 'Feb',
      value: 31
  }, {
      name: 'Mar',
      value: 34
  }, {
      name: 'Apr',
      value: 30
  }, {
      name: 'May',
      value: 34
  }, {
      name: 'Jun',
      value: 38
  }],
  name: 'Component C'
}];

$('#area-example').chart({type: 'line', dataset: dataset});

Keyboard Shortcuts

  • Tab: You can tab into the chart area and through the legend values as each has a focus state.
  • Enter/Space: Will select the area the corresponds with the focus'd legend item.

Upgrading from 3.X
