Settings

Name Type Description
componentAPI string If defined, becomes the definitive way to call methods on parent component.
type string Different types of pagers: list, table and more
position string Can be on 'bottom' or 'top'.
activePage number Start on this page
hideOnOnePage boolean If true, hides the pager if there is only one page worth of results.
source Function Call Back Function for Pager Data Source
pagesize number Can be calculated or a specific number
pagesizes array Array of numbers of the page size selector
showPageSizeSelector boolean If false will not show page size selector
indeterminate boolean If true will not show anything that lets you go to a specific page.

Code Example

The auto initializer will search for ul elements with a paginated class and add a pager to them. You can add the option data-options="{'pagesize': 10}" to set the page size desired. For listview and datagrid this is built in to those components.

Accessibility

  • Since the buttons are arrows, we add an audible span with text like “next”, “previous” so the user will now what these buttons are for
  • icons have role=“presentation” aria-hidden=“true” focusable=“false”
  • select page has aria-selected = “true”
  • disabled pages (if there) have aria-disabled=“true”
