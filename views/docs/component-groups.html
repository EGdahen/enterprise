Navigating an Application

Last Changed: May 1st, 2015

Application menus allow users to navigate through all available pages within an application. If an application supports multiple open pages, users can then use module tabs to navigate between all open pages.

The type of application menu you use depends on the size and complexity of your application.

  • The fly-out application menu works best for more complex applications, and has more flexibility to accomodate multiple levels of navigation. However, it only provides visibility into the currently selected option.
  • The mega menu works best for simpler applications, or for role-specific navigation. It works best with only 2 levels of navigation, and provides visibility into all the options within a specific category. You can also include action buttons for the most commonly performed actions directly on the mega menu.

Regardless of which approach you choose, the application menu should be a fall back method of navigating. Ideally, applications will be organized by role and task, with role-based home pages serving as the primary entry point and main method of navigation. Users should only need to access an application menu when they are performing infrequent tasks that are not part of their regular role.

In addition to menu navigation, in certain applications users can open multiple home pages, workbenches, or forms at one time. For these applications, use module tabs to provide navigation among all open pages.
